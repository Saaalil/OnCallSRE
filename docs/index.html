Agents

Eight specialized AI agents

Each with a distinct role in the incident lifecycle, powered by Gemini 2.5 Pro.

0

Proactive Monitoring Agent

agents/monitoring_agent.py

Runs every 10 minutes via APScheduler. Fetches the latest telemetry and asks Gemini: "Are there leading indicators of a failure?" If an anomaly is predicted, it autonomously triggers the full incident workflow before the system goes down.

Proactive
1

Alert Intake Agent

agents/alert_agent.py

Parses incoming webhook payloads (GitLab pipeline failures, Prometheus alerts). Uses Gemini to classify severity (P1–P4), identify the affected service, and produce a structured alert summary for downstream agents.

Reactive
2

Observability Agent

agents/observability_agent.py

Interfaces with the MCP Server to pull Prometheus metrics and Grafana dashboards. If the MCP connection fails, it gracefully falls back to a rich simulated Prometheus payload for demo purposes.

Reactive
3

Root Cause Analysis Agent

agents/rca_agent.py

Acts as a Manager Agent. Assesses incident complexity and delegates work to two specialized subagents: a Code Analysis Subagent and an Infrastructure Subagent. It then synthesizes their findings into a cohesive root cause hypothesis.

Manager + Subagents
4

Fix Generation Agent

agents/fix_agent.py

Receives the root cause hypothesis and the original code context. Uses Gemini 2.5 Pro to generate a complete, production-grade code patch (e.g., a memory leak fix, a config rollback, a circuit breaker implementation).

Reactive
5

Validation Agent

agents/validation_agent.py

Runs simulated DevSecOps checks on the generated patch: static analysis, dependency vulnerability scanning, unit test validation, and code style compliance. Produces a pass/fail report.

Reactive
6

Deployment Agent

agents/deployment_agent.py

Interfaces with the GitLab MCP Server to create a Merge Request containing the validated patch. Returns the MR URL so the on-call team can review and merge with one click.

MCP Integration
7

Postmortem Agent

agents/postmortem_agent.py

Generates a comprehensive markdown incident postmortem including: timeline of events, root cause analysis, fix applied, blast radius, lessons learned, and preventive recommendations.

Reactive
Workflow

Incident response pipeline

Follow the data as it flows through the autonomous healing pipeline.

01

Trigger

A GitLab webhook fires, the proactive scan detects an anomaly, or the operator clicks the dashboard button.

02

Alert Triage

Gemini classifies severity, maps affected services, and structures the alert for downstream processing.

03

Metric Collection

Prometheus metrics, error rates, CPU/memory trends, and active alerts are gathered via MCP or fallback.

04

Root Cause Analysis

The RCA Manager spins up Code Analysis and Infrastructure Subagents, then synthesizes findings.

05

Patch Generation

Gemini writes a production-grade code fix targeting the specific root cause identified.

06

Validate & Deploy

The patch is security-checked, validated, and a Merge Request is opened on GitLab via MCP.

07

Postmortem

A full incident report is auto-generated with timeline, analysis, fix, and prevention steps.

Codebase

Every file explained

Root

main.py Entrypoint

The Flask web server. Serves the dashboard UI at /, handles GitLab webhooks at /webhook/gitlab, exposes the proactive scan at /api/proactive-check, and initializes the APScheduler background cron job that runs the Proactive Monitoring Agent every 10 minutes.

requirements.txt Dependencies

Python dependencies: google-genai, google-adk, flask, python-dotenv, requests, mcp, apscheduler.

Dockerfile Container

Docker container definition for deploying the platform to Google Cloud Run or any containerized environment.

.env Secrets

Environment variables file (git-ignored). Contains GEMINI_API_KEY for authenticating with the Google GenAI SDK.

agents/

llm_helper.py Core LLM

Centralized Gemini integration module. All agents call ask_gemini(system_prompt, user_prompt) from this file. It initializes the google.genai client with the API key and sends requests to gemini-2.5-pro.

root_agent.py Orchestrator

The main orchestrator. Chains all 7 agents sequentially: Alert → Observability → RCA → Fix → Validation → Deployment → Postmortem. Returns a unified context dictionary consumed by the frontend.

monitoring_agent.py Proactive

The proactive scanner. Fetches metrics, sends them to Gemini asking for leading failure indicators, and returns a synthetic alert payload if an anomaly is detected.

alert_agent.py Agent

Parses raw webhook/alert payloads. Uses Gemini to classify severity and extract the affected service name, pipeline ID, and failure details.

observability_agent.py Agent

Calls the MCP server for Prometheus metrics. On failure, injects a rich simulated payload containing memory leak trends, CPU spikes, OOM alerts, and Grafana dashboard links.

rca_agent.py Manager

Implements the Subagent Delegation Pattern. Spins up a Code Analysis Subagent and an Infrastructure Subagent (separate Gemini calls with focused prompts), then synthesizes their findings into a unified root cause hypothesis.

fix_agent.py Agent

Generates a production-grade code patch using Gemini, targeting the specific fault identified by the RCA Agent.

validation_agent.py Agent

Runs simulated DevSecOps validation: SAST analysis, dependency vulnerability scanning, unit test coverage, and code style compliance checks.

deployment_agent.py MCP

Creates a GitLab Merge Request via the MCP protocol, attaching the validated patch and the RCA summary as the MR description.

postmortem_agent.py Agent

Generates a comprehensive markdown incident postmortem report including timeline, blast radius, fix details, and preventive recommendations.

tools/

gitlab_mcp_tools.py MCP Client

MCP client implementation using the Python mcp SDK. Connects to the GitLab MCP Server via stdio transport (npx mcp-remote). Contains call_gitlab_tool_sync()—the synchronous wrapper used by all agents. Currently mocked for hackathon demo.

metrics_mcp_tools.py Metrics

Utility module for fetching infrastructure metrics. Designed to interface with Prometheus/Grafana endpoints or MCP metric tools.

templates/ & static/

templates/index.html UI

The main dashboard HTML. Split-screen layout: agent result cards on the left, a live terminal stream on the right. Contains two action buttons for reactive and proactive triggers.

static/style.css Styles

Dark-mode glassmorphic CSS with vibrant gradients, blur effects, smooth micro-animations, and a macOS-style terminal window aesthetic.

static/app.js Logic

Frontend JavaScript. Handles button click events, sends fetch requests to the Flask API, renders mock terminal logs with timed delays, and implements a typewriter animation for Gemini response rendering.

Integration

Deploy in your infrastructure

1

Clone & Install

git clone https://github.com/Saaalil/OnCallSRE.git
cd OnCallSRE
pip install -r requirements.txt
2

Configure Environment

# Create a .env file in the root
GEMINI_API_KEY=your_google_genai_api_key

Get your API key from Google AI Studio.

3

Connect GitLab MCP

To enable live MCP (creating real Merge Requests), update tools/gitlab_mcp_tools.py:

# Uncomment the real MCP client code
# and remove the mock responses.
# Requires: npx -y mcp-remote
# Endpoint: https://gitlab.com/api/v4/mcp
4

Connect Prometheus

To pull real infrastructure metrics, update agents/observability_agent.py to point to your Prometheus endpoint instead of the fallback mock payload.

# Replace the mock data with:
# requests.get("http://your-prometheus:9090/api/v1/query")
5

Configure GitLab Webhook

In your GitLab project, go to Settings → Webhooks and point it to:

https://your-deployed-url/webhook/gitlab
# Trigger: Pipeline events
6

Deploy to Cloud Run

gcloud builds submit --tag gcr.io/PROJECT/oncallsre
gcloud run deploy oncallsre \
  --image gcr.io/PROJECT/oncallsre \
  --platform managed \
  --allow-unauthenticated
